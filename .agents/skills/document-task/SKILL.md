# Document Task

## Load context

Read `AGENTS.md`, the Docs role description, task packet, handoff, QA/review
reports, and the accepted implementation.

## Document

1. Identify affected audiences: users, developers, operators, or contributors.
2. Update the smallest authoritative documents.
3. Keep commands executable and contract examples synchronized with code.
4. Explain migration, compatibility, privacy, and known limitations where relevant.
5. Link to source contracts instead of duplicating unstable detail.
6. Run documentation/link checks when available.
7. Report changed documents and recommend documentation impact `resolved`.

## Boundaries

Do not describe unaccepted behavior as complete. Do not include credentials,
private novel text, local paths, or speculative functionality.
